### CWE Detection Implementation Details

**CWE-787: Out-of-bounds Write**
- Patterns detected:
  * strcpy, strcat without length checks
  * sprintf without size validation
  * gets() usage (banned function)
  * memcpy with unchecked size
  * Buffer allocation mismatches
- Languages: C, C++, Python (ctypes), Java (JNI)
- Context analysis: 5-line window for validation checks

**CWE-125: Out-of-bounds Read**
- Patterns detected:
  * Array access without bounds check
  * Pointer arithmetic without validation
  * Buffer underflow conditions
- Languages: C, C++, Rust (unsafe), Python
- Dataflow tracking for array index sources

**CWE-77: Command Injection Special Elements**
- Patterns detected:
  * Shell metacharacters: ; | & $ ` < > ( )
  * Command chaining without sanitization
  * Subprocess with shell=True
  * os.system() with user input
- Languages: Python, JavaScript, PHP, Ruby, Bash
- Taint analysis from user input sources

**CWE-269: Improper Privilege Management**
- Patterns detected:
  * setuid(0), seteuid(0)
  * Running as root without dropping privileges
  * sudo command execution
  * Administrative API access without checks
- Languages: Python, JavaScript, Java, Go, C
- Context: Privilege escalation patterns

**CWE-863: Incorrect Authorization**
- Patterns detected:
  * Missing authorization on DELETE/UPDATE endpoints
  * IDOR (Insecure Direct Object Reference)
  * Resource access without ownership validation
  * Missing role checks on admin functions
- Languages: Python, JavaScript, Java, PHP
- Framework-specific patterns (Express, Django, Spring)

**CWE-276: Incorrect Default Permissions**
- Patterns detected:
  * chmod 777, chmod 666
  * World-writable files (0o666)
  * Containers running as root
  * Missing USER directive in Dockerfile
  * Overly permissive ACLs
- Languages: Python, JavaScript, Bash, Dockerfiles
- File permission analysis
